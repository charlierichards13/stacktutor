# StackTutor Design System

## Brand

StackTutor is a polished AI code review trainer for students learning to debug.

## Tagline

Learn to debug. Don't just get the answer.

## Design Direction

StackTutor should feel like a premium developer education tool, not a generic AI chatbot.

The visual style should be:

- Dark-first
- Clean and modern
- Code/editor inspired
- Calm, focused, and trustworthy
- Professional enough for recruiters
- Approachable enough for students

## Color Palette

### Backgrounds

- App background: `#020617`
- Elevated surface: `#0F172A`
- Card surface: `#111827`
- Border: `#1E293B`

### Text

- Primary text: `#F8FAFC`
- Secondary text: `#CBD5E1`
- Muted text: `#64748B`

### Brand Colors

- Primary: `#7C3AED`
- Primary light: `#A78BFA`
- Accent cyan: `#22D3EE`
- Success: `#22C55E`
- Warning: `#F59E0B`
- Error: `#EF4444`

## Typography

Use clean sans-serif typography for the UI.

Use monospace accents for:

- Code snippets
- Language labels
- Technical badges
- Review mode labels

## UI Principles

- Every screen should have clear hierarchy.
- Avoid random one-off styles.
- Use reusable components.
- Empty states, loading states, and errors must look intentional.
- AI output should be structured, not a wall of text.

## Product Positioning

StackTutor is not a cheating app.

It is a guided debugging trainer that gives students:

- Hints
- Explanations
- Test cases
- Complexity analysis
- Security review feedback
- Step-by-step reasoning
